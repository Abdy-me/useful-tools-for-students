🎨 TITLE
GIT – Version Control System
Working Smarter with Code Management

🟦 SECTION 1: What is Git?
Git is a version control system used by developers to track changes in their code.

It helps teams and individuals manage projects efficiently.

📌 Key Idea:
👉 Git saves every change made in a project.

🟩 SECTION 2: Why Use Git?
Benefits of Git:
Tracks changes in code history

Allows rollback to previous versions

Supports teamwork and collaboration

Works offline

Helps manage large projects easily

🟨 SECTION 3: How Git Works
Git works by saving snapshots of your project.

Basic Flow:
Write Code → Save Changes → Commit → Track History
📌 Each commit = a saved version of your project

🟥 SECTION 4: Important Git Commands
Common Commands:
git init        # Start a repository
git add .       # Add files
git commit -m "message"   # Save changes
git status      # Check changes
git log         # View history
🟪 SECTION 5: Git in Teamwork
Git allows multiple developers to work together.

How it helps teams:
Everyone works on different features

No code overwriting

Easy merging of changes

🟫 SECTION 6: Branching Concept
What is a Branch?
A branch is a separate workspace in Git.

main
 ├── feature-login
 ├── feature-profile
👉 Each feature is developed separately

⚫ FOOTER SECTION
💡 Summary
Git helps developers:

Track code changes

Work in teams

Manage projects safely
